# Free Data Sources

This project ships with **synthetic sample data** (`data/raw/*.csv`, generated by
`ingestion/generate_sample_data.py`) so the full pipeline runs out of the box
with zero API keys. The sample data is *statistically realistic* (built from
tiered Elo ratings and Elo-based match simulation) but is **not real match
history** — replace it with one of the free sources below before trusting any
output for real analysis.

| Data need                     | Free source                                                                 | Format |
|--------------------------------|------------------------------------------------------------------------------|--------|
| International match results    | Kaggle: "International football results from 1872 to 2024" (martj42)        | CSV download |
| Club/National Elo ratings       | http://clubelo.com/ (CSV export endpoints, no key required)                 | CSV |
| National team Elo ratings       | https://www.eloratings.net/ (World Football Elo Ratings, exportable)        | CSV/HTML |
| FIFA Rankings                   | https://www.fifa.com/fifa-world-ranking (official, downloadable tables)     | HTML/CSV |
| World Cup historical results    | Kaggle: "FIFA World Cup" datasets, or https://github.com/jfjelstul/worldcup | CSV |
| Squad value / age (optional)    | https://www.transfermarkt.com (public pages; scrape responsibly, check ToS) | HTML |

## Swapping in real data

1. Download the CSVs above into `data/raw/`, matching the column names used in
   `ingestion/generate_sample_data.py` (see the docstrings for the exact
   schema each downstream module expects: `teams.csv`, `historical_matches.csv`).
2. Delete or rename the generated sample files so they are not overwritten.
3. Re-run `python train_model.py` to retrain on the real data.

No paid API keys are required anywhere in this project.

## Keeping predictions current with live results

As real matches are played (qualifiers, warm-up friendlies, or the
tournament itself), feed the results back in so the model and simulation
reflect actual current form -- see `ingestion/live_updates.py`.

**Option A -- manual CSV (no signup, always free):**
1. Copy `data/raw/live_results_manual.csv.example` to
   `data/raw/live_results_manual.csv` and add finished-match rows.
2. In the app's "Live Updates" tab, click **Refresh from manual CSV** (or run
   `python -c "from ingestion.live_updates import refresh_from_manual_csv; print(refresh_from_manual_csv().as_markdown())"`).
3. This updates each team's Elo rating (standard Elo formula), appends the
   results to `historical_matches.csv`, and rebuilds features + the
   "current form" snapshot used by the simulator -- all in one step.

**Option B -- football-data.org free tier (free API key, covers the World Cup):**
1. Get a free key at https://www.football-data.org/client/register (free
   forever for World Cup + 11 other competitions, 10 requests/minute).
2. Set it as an environment variable: `export FOOTBALL_DATA_API_KEY=...`
3. Use the app's "Live Updates" tab (or `refresh_from_api()`) to fetch and
   apply newly finished matches automatically.

Both paths are additive and safe to re-run: duplicate match rows are
de-duplicated by (date, home_team, away_team), so re-fetching or re-running
never double-counts a result.
